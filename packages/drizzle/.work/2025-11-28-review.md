- Description: The equality query test now filters by both encrypted age AND test_run_id (line 287), which is good. However, the assertion only checks `expect(data).toHaveLength(1)` without verifying it's actually the correct inserted record. Other tests use ID-based lookups which are more precise.

- Description: The afterAll cleanup only runs if `insertedIds.length > 0`. If a test fails during insertion before pushing to insertedIds, stale data could accumulate. The beforeAll cleanup mitigates this with pattern-based deletion, but there's a window for race conditions.
- Location: packages/protect/__tests__/supabase.test.ts:55-66
- Action: Consider wrapping each test's insert operations in try-finally blocks to ensure IDs are tracked even on partial failures, or document that beforeAll cleanup is the primary defense against stale data.
